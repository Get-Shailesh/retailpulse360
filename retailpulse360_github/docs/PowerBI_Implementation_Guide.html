⚙️ Step 1 — Project Setup & Data Connections

01

Connect to PostgreSQL

Open Power BI Desktop → Get Data → PostgreSQL. Enter server address, database name retailpulse360. Import mode for data under 1GB, DirectQuery for live refresh.

02

Import CSV Outputs

Also import Python-generated CSVs: rfm_scores.csv and churn_scores.csv from the data folder. These contain ML model outputs.

03

Build the Data Model

In Model view, connect: fact_transactions → dim_stores, dim_products, dim_customers, dim_date. Star schema — all relationships Many-to-One.

04

Link ML Outputs

Join rfm_scores to dim_customers on customer_id. Join churn_scores to dim_customers on customer_id.

Set the dim_date table as your date table (right-click → Mark as Date Table). This enables Power BI's time intelligence functions like SAMEPERIODLASTYEAR.

📐 Step 2 — Core DAX Measures

Create a dedicated _Measures table (blank query) and store all measures there. Never put measures in fact tables.

MEASURE — Total Revenue
Total Revenue = SUM(fact_transactions[total_revenue])
MEASURE — Gross Profit Margin %
GP Margin % = 
DIVIDE(
    SUM(fact_transactions[gross_profit]),
    SUM(fact_transactions[total_revenue]),
    0
)
MEASURE — Year-over-Year Growth
YoY Revenue Growth = 
VAR CurrentYear = [Total Revenue]
VAR PrevYear = CALCULATE([Total Revenue], SAMEPERIODLASTYEAR(dim_date[date_id]))
RETURN
DIVIDE(CurrentYear - PrevYear, PrevYear, 0)
MEASURE — Rolling 3-Month Average
Rolling 3M Revenue = 
CALCULATE(
    [Total Revenue],
    DATESINPERIOD(dim_date[date_id], LASTDATE(dim_date[date_id]), -3, MONTH)
)
MEASURE — Churn Risk Customer Count
High Risk Customers = 
CALCULATE(
    DISTINCTCOUNT(churn_scores[customer_id]),
    churn_scores[churn_risk_label] = "High Risk"
)
MEASURE — Average Order Value
Avg Order Value = 
DIVIDE(
    [Total Revenue],
    DISTINCTCOUNT(fact_transactions[transaction_id]),
    0
)
MEASURE — Revenue Target (Dynamic)
Revenue vs Target = 
VAR Target = [Total Revenue] * 1.10  -- 10% growth target
RETURN
DIVIDE([Total Revenue], Target, 0)

🤖 Step 4 — Enabling AI Visuals

Smart Narrative (Auto-generated Insights)

Insert → Smart Narrative. Drag [Total Revenue], [YoY Revenue Growth], and [High Risk Customers] into the field well. Power BI will auto-generate sentences like "Revenue grew 8% YoY. 1,985 customers are at high churn risk." Customise the template in the Format pane.

Sales Forecast (Built-in Analytics)

Create a line chart with dim_date[date_id] on X-axis and [Total Revenue] on Y-axis. In the Analytics pane → Forecast → set Units = Months, Forecast length = 3, Confidence interval = 95%. This adds the shaded forecast cone directly to the chart.

Anomaly Detection

In the same line chart, Analytics pane → Find Anomalies → Sensitivity = 85%. Power BI will automatically highlight anomalous data points with red dots and explain them in a tooltip: "Revenue was 23% below expected on this date."

Key Influencers Visual

Insert → Key Influencers. Set Analyze = [Total Revenue]. Add Explain by: dim_stores[region], dim_products[category], dim_customers[loyalty_tier], rfm_scores[segment]. Power BI uses ML to identify which factors most drive revenue changes.

Decomposition Tree

Insert → Decomposition Tree. Analyze = [Total Revenue]. Add Explain by: Region → Store Type → Category → Subcategory → Product. Users can interactively drill down to find the exact root cause of revenue differences.

Enable the "Preview features" in Power BI settings to access Copilot — it can generate DAX measures from natural language and create report pages automatically from your data model.

🔒 Step 5 — Row-Level Security

Regional Manager RLS

In Modeling → Manage Roles → Create role "RegionalManager". Add DAX filter to dim_stores:

[region] = USERPRINCIPALNAME() -- or use a lookup table mapping email → region

For a mapping table approach, create a user_region_map table (email, region) and filter:

dim_stores[region] IN
    CALCULATETABLE(
        VALUES(user_region_map[region]),
        user_region_map[email] = USERPRINCIPALNAME()
    )
Always test RLS with "View as Role" before publishing. RLS is enforced in Power BI Service — in Desktop, all data is visible unless you explicitly test the role.

🚀 Step 6 — Publishing & Scheduled Refresh

01

Publish to Power BI Service

File → Publish → Select your workspace. The .pbix file uploads to the cloud. All team members with workspace access can view the report.

02

Configure Data Gateway

If using DirectQuery or on-premise SQL: install On-premises Data Gateway on the server. Connect in Power BI Service → Settings → Manage Gateways.

03

Set Scheduled Refresh

Dataset Settings → Scheduled Refresh → Daily at 6:00 AM. For CSV files from Python pipeline, schedule the Python script to run at 5:30 AM first.

04

Share & Embed

Share the report URL with stakeholders, or embed in a Teams tab or SharePoint page. Use "Publish to web" only for non-sensitive public data.
